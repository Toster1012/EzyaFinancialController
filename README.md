# Ezya - Калькулятор финансов

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Приложение для отслеживания личных финансов на Android. Ведите учёт доходов и расходов по категориям, анализируйте траты с помощью диаграмм и просматривайте историю периодов.

## Возможности

- Бюджетирование по периодам (неделя, месяц, год)
- Категории доходов и расходов с пользовательскими иконками
- Столбчатые диаграммы для визуального анализа трат
- Календарь транзакций с разбивкой по дням
- История завершённых периодов с архивными результатами
- Светлая и тёмная тема
- Русский и английский языки интерфейса

## Сборка из архива

### Что потребуется

Перед началом установите на компьютер следующее:

- [Android Studio](https://developer.android.com/studio) — версия Hedgehog (2023.1.1) или новее
- JDK 17 - устанавливается вместе с Android Studio автоматически
- Android SDK с API 24 или выше - устанавливается через Android Studio
- Минимум 8 ГБ оперативной памяти и 10 ГБ свободного места на диске
- Стабильное интернет-соединение для первой синхронизации зависимостей

---

### Шаг 1 - Распаковка архива

Скачайте архив проекта и распакуйте его в удобное место без кириллицы и пробелов в пути, например:

```
C:\Projects\Ezya\
```

После распаковки внутри папки должна находиться следующая структура:

```
Ezya/
├── app/
├── gradle/
├── build.gradle
├── settings.gradle
├── gradle.properties
├── gradlew
└── gradlew.bat
```

Если вместо этого видите ещё одну вложенную папку — откройте её, структура должна быть внутри.

---

### Шаг 2 - Открытие проекта в Android Studio

1. Запустите Android Studio
2. На стартовом экране нажмите **Open**
3. В диалоге выбора найдите папку `Ezya` - ту, внутри которой лежит `build.gradle` - и нажмите **OK**
4. Дождитесь завершения индексации. При первом открытии это может занять 3–10 минут

---

### Шаг 3 - Синхронизация Gradle

Android Studio автоматически начнёт загрузку зависимостей. В нижней панели появится прогресс-бар с текстом **Gradle: sync**.

Если синхронизация не запустилась автоматически, нажмите **File -> Sync Project with Gradle Files**.

Дождитесь появления в нижней панели сообщения **Gradle sync finished**.

Если появляется ошибка про отсутствующий SDK, откройте **File -> Project Structure -> SDK Location** и укажите путь. Обычно SDK находится здесь:

```
Windows:  C:\Users\ВашеИмя\AppData\Local\Android\Sdk
Mac:      /Users/ВашеИмя/Library/Android/sdk
Linux:    /home/ВашеИмя/Android/Sdk
```

---

### Шаг 4 - Сборка проекта

Нажмите **Build -> Make Project** или используйте сочетание клавиш:

```
Windows / Linux:  Ctrl + F9
Mac:              Cmd + F9
```

В нижней панели Build должно появиться сообщение **BUILD SUCCESSFUL**.

---

### Шаг 5 - Запуск на устройстве

**На физическом устройстве Android:**

1. На телефоне откройте **Настройки -> О телефоне** и нажмите на строку **Номер сборки** семь раз подряд - появится уведомление «Вы стали разработчиком»
2. Вернитесь в Настройки, откройте появившийся раздел **Для разработчиков** и включите **Отладка по USB**
3. Подключите телефон к компьютеру кабелем USB
4. На телефоне появится запрос - разрешите отладку с этого компьютера
5. В Android Studio в верхней панели инструментов выберите ваше устройство из выпадающего списка рядом с кнопкой запуска
6. Нажмите зелёную кнопку запуска или **Shift + F10** на Windows / **Ctrl + R** на Mac

**На эмуляторе:**

1. Откройте **Device Manager** через иконку в правой вертикальной панели Android Studio
2. Нажмите **Create Device**
3. Выберите любую модель телефона из списка, например Pixel 6, нажмите **Next**
4. Выберите образ системы с API 24 или выше. Если образ не скачан - нажмите на иконку загрузки рядом с ним и дождитесь завершения
5. Нажмите **Next -> Finish**
6. Запустите эмулятор кнопкой с треугольником в Device Manager и дождитесь загрузки системы
7. Нажмите кнопку запуска в Android Studio

---

### Сборка APK для ручной установки

Если нужен APK файл для установки без Android Studio:

1. Нажмите **Build -> Build Bundle(s) / APK(s) -> Build APK(s)**
2. Дождитесь завершения - в нижнем правом углу появится уведомление
3. Нажмите **locate** в уведомлении или найдите файл вручную по пути:

```
Ezya\app\build\outputs\apk\debug\app-debug.apk
```

Перенесите APK на телефон через USB, мессенджер или облако и откройте для установки.

Если установка заблокирована, откройте **Настройки -> Безопасность** и разрешите установку из неизвестных источников. На Android 8 и выше это разрешение запрашивается при открытии конкретного APK файла.

---

### Возможные проблемы при сборке

**Gradle sync failed** - проверьте подключение к интернету. При первом запуске Gradle скачивает около 200 МБ зависимостей.

**SDK location not found** - укажите путь к SDK вручную через **File -> Project Structure -> SDK Location**.

**Устройство не отображается в списке** - переустановите драйверы USB для вашего телефона или попробуйте другой кабель и другой порт USB.

**BUILD FAILED** - нажмите **Build -> Clean Project**, подождите завершения, затем повторите **Build -> Rebuild Project**.

**Ошибка Duplicate resources** - убедитесь что в папке `res/values/` нет файла `styles.xml` или что в нём нет стилей `DarkAlertDialog` и `YellowButton` - они должны быть только в `themes.xml`.

## Как пользоваться

### 1. Первый запуск

На экране приветствия нажмите «Начать» для настройки. Вы пройдёте двухшаговую конфигурацию бюджета. Нажмите кнопку «?» в левом верхнем углу чтобы открыть встроенную справку.

### 2. Настройка категорий доходов

Выберите длину периода - неделя, месяц или год. Затем добавьте источники дохода. Для каждой категории введите название, выберите иконку и укажите планируемую сумму.

### 3. Настройка категорий расходов

После подтверждения доходов добавьте категории расходов. Общая сумма расходов не может превышать плановый доход. Остаток бюджета отображается в реальном времени. Нажмите на существующую категорию чтобы отредактировать её.

### 4. Главный экран

Главный экран содержит две вкладки - Доходы и Расходы - между которыми можно переключаться свайпом. Каждая вкладка содержит столбчатую диаграмму транзакций по категориям и список истории операций.

### 5. Внесение изменений

Нажмите «Внести изменения» чтобы записать реальный доход или расход. Тип операции выбирается автоматически в зависимости от активной вкладки.

### 6. Календарь

Кнопка в левом верхнем углу главного экрана открывает календарь. Дни с транзакциями отмечены точками. Нажмите на отмеченный день чтобы увидеть список всех операций за этот день с итогами и балансом.

### 7. Завершение периода

По истечении периода экран итогов открывается автоматически. Также можно завершить период досрочно. В итогах отображаются общий доход, расход и экономия или перерасход по каждой категории.

### 8. Новый период

После просмотра итогов нажмите «Начать новый период». Категории сохраняются - их не нужно настраивать заново.

### 9. История

Нажмите «История» на главном экране чтобы просмотреть все завершённые периоды с датами начала и окончания. Внутри каждого периода доступны диаграммы и кнопка «Итоги».

### 10. Настройки

Доступны через иконку шестерёнки на главном экране: смена темы, смена языка, очистка истории, оценка приложения.
